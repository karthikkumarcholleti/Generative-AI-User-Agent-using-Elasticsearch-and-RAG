┌───────────────────────────────────────────────────────────────────────────────┐
│ ✅ CASE 1: ELASTICSEARCH ALREADY INSTALLED (YOUR CASE)                       │
└───────────────────────────────────────────────────────────────────────────────┘

STEP 1: Kill All Running Terminals/Processes
1. Kill existing FHIR processes:
   pkill -f "uvicorn" && pkill -f "python server.py"

2. Wait for ports to be free:
   sleep 3

STEP 2: Start ElasticSearch
1. Find your ElasticSearch:
   find / -name "elasticsearch" -type f 2>/dev/null | grep bin/elasticsearch

2. Check if running:
   curl -k -u elastic:P@ssw0rd https://localhost:9200

3. If not running, start it:
   cd /path/to/your/elasticsearch
   ./bin/elasticsearch -d
   sleep 10

4. Verify connection:
   curl -k -u elastic:P@ssw0rd https://localhost:9200

STEP 3: Start FHIR Application
1. Navigate to project and activate environment:
   cd /home/kchollet/LLM_UA/FHIR_LLM_UA
   source venv/bin/activate

2. Start backend server:
   cd backend
   python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000 &

3. Start frontend server:
   cd ../frontend
   python server.py &

4. Open browser:
   xdg-open http://localhost:5173

5. Index patient data:
   - Open browser to http://localhost:5173
   - Click "🔄 Index All Patients" button in sidebar

┌───────────────────────────────────────────────────────────────────────────────┐
│ 🐘 CASE 2: ELASTICSEARCH NOT INSTALLED (FRESH INSTALLATION)                  │
└───────────────────────────────────────────────────────────────────────────────┘

STEP 1: Download and Install ElasticSearch
1. Download ElasticSearch:
   wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-8.14.0-linux-x86_64.tar.gz

2. Extract:
   tar -xzf elasticsearch-8.14.0-linux-x86_64.tar.gz
   cd elasticsearch-8.14.0/

3. Generate SSL certificate:
   ./bin/elasticsearch-certutil cert -out config/elastic-certificates.p12 -pass ""

4. Configure security (edit config/elasticsearch.yml):
   nano config/elasticsearch.yml
   # Add these lines:
   xpack.security.enabled: true
   xpack.security.transport.ssl.enabled: true
   xpack.security.transport.ssl.keystore.path: elastic-certificates.p12
   xpack.security.transport.ssl.keystore.password: ""
   xpack.security.transport.ssl.verification_mode: certificate

5. Start ElasticSearch:
   ./bin/elasticsearch -d

6. Set password:
   ./bin/elasticsearch-reset-password -u elastic -i
   # Enter: P@ssw0rd

7. Verify it's working:
   curl -k -u elastic:P@ssw0rd https://localhost:9200

STEP 2: Start FHIR Application
1. Kill existing processes:
   pkill -f "uvicorn" && pkill -f "python server.py"

2. Start application:
   cd /home/kchollet/LLM_UA/FHIR_LLM_UA
   source venv/bin/activate
   cd backend && python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000 &
   cd ../frontend && python server.py &
   xdg-open http://localhost:5173

3. Index patient data:
   - Open browser to http://localhost:5173
   - Click "🔄 Index All Patients" button

Problem: "Address already in use" (Port 8000, 5173, or 9200)
Solution:
  pkill -f "uvicorn" && pkill -f "python server.py" && pkill -f "elasticsearch"
  sleep 3
  netstat -tlnp | grep -E "(8000|5173|9200)" || echo "Ports are free"
  [Then restart as normal]

Problem: "Unable to connect" in browser
Solution:
  curl -s http://localhost:5173 | head -c 50
  [If fails, restart frontend:]
  cd /home/kchollet/LLM_UA/FHIR_LLM_UA/frontend
  python server.py &

Problem: Backend not responding
Solution:
  curl -s http://localhost:8000/health
  [If fails, restart backend:]
  cd /home/kchollet/LLM_UA/FHIR_LLM_UA/backend
  source ../venv/bin/activate
  python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000 &

Problem: ElasticSearch connection failed
Solution:
  # Check if ElasticSearch is running:
  curl -k -u elastic:P@ssw0rd https://localhost:9200
  # If fails, start ElasticSearch:
  cd /path/to/elasticsearch-8.14.0
  ./bin/elasticsearch -d
  sleep 10
  # Verify connection:
  curl -k -u elastic:P@ssw0rd https://localhost:9200

Problem: Chat agent shows "ElasticSearch not connected"
Solution:
  # Check ElasticSearch status:
  curl http://localhost:8000/chat-agent/status
  # If ElasticSearch is down, restart it:
  cd /path/to/elasticsearch-8.14.0
  ./bin/elasticsearch -d
  # Wait and re-index data:
  curl -X POST http://localhost:8000/chat-agent/index-all-patients

Problem: "No data available" in chat agent
Solution:
  # Check if data is indexed:
  curl http://localhost:8000/chat-agent/status
  # If not indexed, click "🔄 Index All Patients" button in UI
  # Or use API:
  curl -X POST http://localhost:8000/chat-agent/index-all-patients

Problem: Check what's running
Solution:
  ps aux | grep -E "uvicorn|server.py|elasticsearch" | grep -v grep
  netstat -tlnp | grep -E "(8000|5173|9200)"

┌───────────────────────────────────────────────────────────────────────────────┐
│ 🚨 COMMON ISSUES & SOLUTIONS                                                  │
└───────────────────────────────────────────────────────────────────────────────┘

Issue: "No data available" in chat agent
Solution: Click "🔄 Index All Patients" button in sidebar

Issue: General medical help not working
Solution: Check if backend is running: curl http://localhost:8000/health

Issue: Charts not displaying
Solution: Check browser console for errors, refresh page

Issue: Patient data not loading
Solution: Verify ElasticSearch is connected: curl http://localhost:8000/chat-agent/status

Issue: ElasticSearch connection timeout
Solution: 
  1. Check if ElasticSearch is running: ps aux | grep elasticsearch
  2. Start ElasticSearch: cd /path/to/elasticsearch-8.14.0 && ./bin/elasticsearch -d
  3. Wait 10 seconds and verify: curl -k -u elastic:P@ssw0rd https://localhost:9200

Issue: Chat agent returns "ElasticSearch not available"
Solution:
  1. Check ElasticSearch status: curl http://localhost:8000/chat-agent/status
  2. If ElasticSearch is down, restart it
  3. Re-index patient data: curl -X POST http://localhost:8000/chat-agent/index-all-patients

Issue: SSL certificate errors with ElasticSearch
Solution:
  1. Use -k flag with curl: curl -k -u elastic:P@ssw0rd https://localhost:9200
  2. Or disable SSL verification in browser (development only)

Issue: Memory issues with ElasticSearch
Solution:
  1. Increase system memory or close other applications
  2. Adjust ElasticSearch heap size in config/jvm.options
  3. Consider running ElasticSearch on a separate machine
